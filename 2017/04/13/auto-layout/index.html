UI设计师初尝 Auto Layout

去年我在淘宝代购了一本由 Simon Ng 写的 iOS 9 App程式設計實力超進化實戰攻略 (Beginning iOS 9 Programming with Swift)，刚买回来看几天就正好要赶项目，就把书丢到一边去了。

直到最近稍微空闲一点了，重新拿起这本书，尝试学习怎样使用 Xcode 和 Swift 去开发 APP。

这本书非常推荐设计师和完全不懂编程的初学者购买，前几章主要介绍 Xcode 的 GUI 操作和实例，跟着范例自己做个会动的页面出来很有成就感，能学到很多。

UI设计师要懂 Auto Layout



我跟着书里第 5 章范例做得 Auto Layout 练习，有居中的标题、居中的按钮，和底部偏右侧对齐的文字。

  • 左，iPhone 4s ，标题和 Hello World 按钮 居中，但屏幕高度不足导致下方文字无法显示。
  • 中，iPhone 5s ，标题和 Hello World 按钮 居中，下方文字偏右，一切正常。
  • 右，iPhone 6s ，标题偏左、按钮偏左、文字位置也不对。

UI 设计师都知道要给工程师的标注稿上需要写明尺寸、间距等描述，iOS 还可以给坐标。但在不同手机尺寸下，给坐标就会出现如上图一样尴尬的情况了。

先来解决不同尺寸下控件位置错位的问题，就要使用到 Auto Layout 了。（暂时先跳过 Portrait 和 Landscape 的页面变化，以及 Universal App 选项，以后有机会再另开一篇讨论。）

对齐与定位



其实如果常用 Sketch 做页面的童鞋看到这 2 个图标大概都能猜得出工程师会怎样设定 Auto Layout 了吧？（虽然工程师都写代码直接处理，但用 GUI 来表达对于设计师们来说总是更容易理解的。）

  • Align：对齐。建立一个对齐的约束条件，如：控件居中对齐。
  • Pin：定位。建立间距约束条件，如：控件离屏幕上、右、下、左的间距。

所以设计师出标注稿给工程师的时候，需要写明对齐方式和定位。比如 A 和 B 之间的距离是多少？居中还是左对齐？某控件如果固定在左下方是不是要标明距离左侧和下方的距离……等等。

实例

因为尺寸不同所以控件错位的情况，可以透过设定 Auto Layout 解决。设好后 Storyboard 会出现几条辅助线。

标题



不管屏幕尺寸变化，标题都希望要居中，且距离顶部的位置要固定。

按钮



按钮设为「水平对齐」、「垂直对齐」。这样不管在哪种萤幕尺寸下它都在正中间。

文字



文字设定距离右边和距离底部的尺寸，不管萤幕尺寸变化，这行文字都会固定在右下角。

Er…这就产生一个新的问题，比如 avocode 和 zeplin 这类不用设计师出标示文件的好用外掛，如何让工程师知道在不同萤幕尺寸下，各元件的位置呢？设计师哪有这么勤劳针对各种萤幕尺寸出一堆原始档。

看是要拼一定点另外写份标示文件，或是干脆放手让工程师处理掉了吧。

延伸思考

Auto Layout 不只能处理各种萤幕尺寸下元件位置的版面问题，Portrait 和 Landscape （手机转横或转直）间的切换也可以用它来处理，甚至 Universal App（iPhone 和 iPad 同一支 app 程式），iPhone 和 iPad 之间的 Responsive 也可以透过 Auto Layout 搞定。

嗯，这就来讨论下 1. 为什么 Landscape 要用 Auto Layout，2. 为什么 Universal App 要用 Auto Layout。

为什么 Landscape 要用 Auto Layout

从使用者的操作，简单分成 2 种可能性。

  1. 手机转横是为了看更大的画面，比如 Safari。
  2. 手机转横会触发不同的功能。

如果是为了看更大的画面，功能元件几乎相同，用 Auto Layout 我觉得很方便省事。

如果会触发不同功能甚至换个介面，那用 Auto Layout 想干嘛？当然是刻个新的 UI 啊！

为什么 Universal App 要用 Auto Layout

同上方的理由，iPad 的界面和 iPhone 的界面会完全一样吗？iPad 屏幕屏幕比 iPhone 大那么多，操作情境、功能复杂度不会一样。

而且，在开启 App 时就判定 App 被装在哪种装置上，读取 iPad 或 iPhone 的程式码，都比用 Auto Layout 干脆实在啊。
